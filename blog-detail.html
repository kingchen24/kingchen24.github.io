理解 Transformer 中的旋转位置编码 (RoPE)：从几何直觉到数学推导

数字花园长期维护中

在处理大语言模型的长文本外推问题时，位置编码的设计往往决定了模型的生死。最近在深夜重读论文，我重新在草稿纸上推演了一遍大名鼎鼎的 RoPE (Rotary Position Embedding)。这篇文章不打算直接堆砌生硬的算法公式，而是想跟以前一样，顺着我的学习笔记，聊聊如何从最纯粹的二维空间几何直觉一步步演进出那套优雅的复数矩阵旋转。

01. 核心思想：我们要解决什么？

传统的绝对位置编码（如 Transformer 原版中的正余弦编码）就像给每个座位贴上固定标签。但在自注意力机制中，我们真正关心的是词与词之间的相对距离。我们希望找到一个函数，当它作用于两个向量时，它们内积的结果里只包含它们的相对位置差 $m - n$。

獭猫的深夜直觉提示：

“如果把词向量看作是夜空中漂浮的星星，绝对编码是记录它们在星图上的绝对坐标；而旋转编码则是让整个星空以某种特定的规律旋转——星星之间的相对夹角和距离保持不变。这种复数空间的旋转，正是数学上最纯粹的浪漫。”

02. 二维空间中的几何旋转

在线性代数中，要让一个二维向量 $\mathbf{x} = [x_1, x_2]^T$ 旋转一个角度 $\theta$，我们只需要让它乘以一个非常经典的旋转矩阵 $R_\theta$：

$$ R_{\theta, m} = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix} $$

RoPE 的精妙之处就在于，它将高维的词向量拆解成无数个两两一组的二维子空间。对于第 $m$ 个位置的词向量，在每个二维子空间里都让它旋转 $m\theta$ 的角度。当两个旋转后的向量进行内积时，旋转矩阵的特性恰好导致绝对位置相减，完美消去了绝对坐标，只剩下了相对距离。

04. 极简 PyTorch 代码演练

下面是我在本地沙盒环境中调通的核心算子代码。这里为了保证长读舒适性，我将代码中的关键注释写得极其通俗，抛弃了教条式的变量命名。

rope_tensor.py (Python) 
import torch

def apply_rotary_emb(x: torch.Tensor, freqs_cis: torch.Tensor) -> torch.Tensor:
    # 1. 将输入的复数特征拆解
    x_complex = torch.view_as_complex(x.float().reshape(*x.shape[:-1], -1, 2))
    
    # 2. 广播频率矩阵以匹配输入的 Batch 和 Head 形状
    freqs_cis = freqs_cis.view(1, x_complex.shape[1], 1, x_complex.shape[-1])
    
    # 3. 核心魔法：直接在复数空间进行哈达玛积（等价于二维矩阵旋转）
    x_rotated = x_complex * freqs_cis
    
    # 4. 重新铺平为实数空间返回
    return torch.view_as_real(x_rotated).flatten(-2)
